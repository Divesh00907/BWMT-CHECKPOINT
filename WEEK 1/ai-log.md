Interaction 1

Prompt:
“I’m learning HTML. Can you explain what <!DOCTYPE html>, <head>, and <body> do? I want to understand the purpose of each element rather than just copying a template.”

Response summary:
The AI explained that:

<!DOCTYPE html> tells the browser to use HTML5.

<head> contains metadata such as title and character encoding.

<body> contains the visible content of the webpage.

What I learned:
I learned that the <head> is not for visible content and that <!DOCTYPE html> ensures the browser renders the page correctly. Understanding structure helps me avoid placing elements in the wrong section.

Did I use the suggestion?
Yes. I reviewed my page and ensured that visible elements were only inside the <body> section.

Interaction 2

Prompt:
“I’m a student learning HTML. Can you review my page for accessibility issues and semantic improvements? Please explain why each suggestion matters.”

Response summary:
The AI suggested:

Adding lang="en" to the <html> tag.

Adding alt text to images.

Replacing generic <div> elements with semantic elements like <main> or <section>.

It explained that these changes improve accessibility, structure, and SEO.

What I learned:
I learned that accessibility is a key part of HTML, not an optional feature. Semantic elements give meaning to content, which helps screen readers and search engines.

Did I use the suggestion?
Yes. I added missing alt attributes and improved semantic structure. I validated the page afterwards.

Interaction 3

Prompt:
“AI suggested replacing a clickable <div> with a <button>. Why is this better?”

Response summary:
The AI explained that a <button> is designed for interaction and is automatically accessible via keyboard navigation, while a <div> has no built-in functionality or accessibility features.

What I learned:
I learned that using the correct element improves usability and accessibility. Just because something works visually does not mean it is correct semantically.

Did I use the suggestion?
Yes. After understanding the reason, I replaced the <div> with a <button> and tested it in the browser.
